# Project Instructions

- When possible, write tests **prior** to writing code.
- Keep the code simple, readable, and easy to debug. Do not over-simplify.
- Ensure code follows the 5 SOLID principles of object oriented programming.
- Include Go Docs for humans to understand the code.
- Run `trunk check --ci` after code changes.
- Commit whenever you complete a feature, fix, or chore. Commits should be about a single goal.
- Keep responses concise.
